## 关键数据流

### 数据流 1: 服务注册

```
用户代码 → register_singleton() → 创建 ServiceDescriptor → 存储到 _services
```

**步骤说明：**

1. 用户调用 `container.register_singleton(Interface, Implementation)`
2. 创建 `ServiceDescriptor` 对象，包含服务类型、实现、生命周期、依赖
3. 将描述符存储到容器的 `_services` 字典中

**代码位置：** `container.py` 第 152-175 行

### 数据流 2: 服务获取（单例）

```
get() → 检查 _services → 检查缓存 → 返回或创建实例
```

**步骤说明：**

1. 用户调用 `container.get(ServiceType)`
2. 从 `_services` 查找 `ServiceDescriptor`
3. 如果是单例且已有缓存实例，直接返回
4. 否则调用 `_resolve_instance()` 创建新实例
5. 缓存并返回实例

**代码位置：** `container.py` 第 284-319 行

### 数据流 3: 依赖解析

```
_resolve_instance() → 遍历 dependencies → 递归 get() → 构造实例
```

**步骤说明：**

1. 遍历服务的 `dependencies` 列表
2. 对每个依赖调用 `container.get()` 递归获取
3. 将获取的依赖实例打包成字典
4. 调用 `implementation(**deps)` 创建服务实例

**代码位置：** `container.py` 第 321-354 行

### 数据流 4: 循环依赖检测

```
get() → 检查 _resolving 集合 → 抛出异常或继续
```

**步骤说明：**

1. 在 `get()` 开始时检查服务是否在 `_resolving` 集合中
2. 如果在集合中，说明存在循环依赖，抛出 `CircularDependencyException`
3. 否则将服务添加到 `_resolving` 集合
4. 服务解析完成后从集合中移除

**代码位置：** `container.py` 第 301-319 行

## 性能考虑

### 瓶颈点

| 阶段 | 潜在瓶颈 | 优化建议 |
|------|---------|---------|
| 服务注册 | 锁竞争 | 批量注册，使用上下文管理器 |
| 服务获取 | 锁竞争 | 使用细粒度锁，减少锁粒度 |
| 依赖解析 | 递归开销 | 减少依赖深度，使用缓存 |
| 实例创建 | 反射开销 | 预热，使用__slots__ |

### 优化策略

1. **预热**：启动时创建所有单例
2. **缓存**：使用服务级别缓存
3. **延迟加载**：按需创建实例
