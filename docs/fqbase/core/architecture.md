## 概述

Core 模块作为 FQBase 的基础设施核心层，采用了分层架构设计，聚合了事件驱动、日志记录、通知推送等核心能力。

## 组件

### 聚合层 (__init__.py)

**用途：** 统一导出子模块 API

**职责：**
- 导入并重新导出各子模块的核心类和函数
- 提供组合使用的统一入口
- 定义模块级别常量

### EventBus 组件

**用途：** 实现发布-订阅模式的事件总线

**职责：**
- 管理事件订阅
- 事件分发
- 事件历史记录

### Logger 组件

**用途：** 统一的日志记录系统

**职责：**
- 日志级别管理
- 日志格式化
- 日志输出目标管理

### Notification 组件

**用途：** 多渠道通知服务

**职责：**
- 通知渠道管理
- 消息发送
- 消息模板渲染

### EventBus Celery 组件

**用途：** EventBus 与 Celery 的集成

**职责：**
- Celery 任务事件绑定
- 任务生命周期事件发布

### NotificationTemplate 组件

**用途：** 通知消息模板

**职责：**
- 模板定义
- 模板渲染
- 模板注册

## 数据流

### 典型数据流 1: 事件驱动通知

```
1. 业务逻辑发布事件
   event_bus.publish(Event('type', data))
       │
       ▼
2. EventBus 查找订阅者
       │
       ▼
3. 触发订阅者回调
   handler(event)
       │
       ├──▶ Logger 记录日志
       │    logger.info(...)
       │
       └──▶ Notification 发送通知
            notifier.send(...)
```

### 典型数据流 2: Celery 任务监控

```
1. Celery Worker 执行任务
       │
       ▼
2. EventBus Celery 拦截任务事件
       │
       ├──▶ task_success 事件
       │     │
       │     ├──▶ Logger 记录
       │     └──▶ Notification 通知
       │
       └──▶ task_failure 事件
             │
             ├──▶ Logger 记录
             └──▶ Notification 告警
```
